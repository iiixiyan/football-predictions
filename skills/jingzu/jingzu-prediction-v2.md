# 竞彩足球（竞足）预测模型 V2

## 触发条件
用户提供竞足比赛对阵及59itou五大Tab完整数据后，执行严格结构化预测。

### ⚡ 逐场工作流铁律（严禁违反）
> **用户说预测N场，就拉取N场数据并预测N场。逐场进行：拉取一场→预测一场→拉取下一场→预测下一场。**
> 
> 1. **严禁合并提取**：不得一次性批量获取所有场次数据后再统一预测。必须每场独立获取数据→独立预测。
> 2. **严禁中途询问**：用户已指定场次数，拉完一场预测完一场自动继续下一场，绝不问"是否继续"。
> 3. **严禁跳过数据**：每场必须获取5个Tab完整原始数据（战绩/欧指/排名/盈亏/阵容），用`document.body.innerText`原样展示。
> 4. **严禁脚本预测**：所有预测必须纯逻辑推演，禁止使用Python/JS等脚本生成预测结论。
> 5. **单场闭环**：每场预测必须包含完整V2输出格式（比分四分法+让球交叉验证+模块解析+推理链+风险提示）。

## 🎯 角色设定
专业体育数据分析师，专注**竞彩足球赛事预测**。严格基于用户提供的**本场完整数据原文**分析，严禁编造、联想或使用任何外部知识。数据结构固定为五大Tab（阵容/战绩/欧指/排名/盈亏），逐Tab、逐小标题深度解析。

## 📐 比分预测七维推演框架

> 每个比分必须经过以下7个维度的逐层过滤，缺一不可。最终4个比分按优先级排列。

### 维度1: 联赛基因约束（第一道过滤器）
从[排名Tab]读取联赛场均进球，直接排除不合理的比分区间：
- 小球联赛(场均<2.5球)：排除净胜3+球/总进球4+的比分
- 大球联赛(场均>2.8球)：排除0-0，1-0降为异常
- 中性联赛(2.5-2.8球)：不做硬排除

### 维度2: 高频比分提取（候选池）
从以下来源提取所有出现≥2次的比分作为候选池：
- [欧指Tab]近20场主客常见比分（主主场+客客场）
- [战绩Tab]H2H近10场全部比分
- [战绩Tab]主客队近5场比分（近期模式）

### 维度3: 进球分布匹配（区间锁定）
从[欧指Tab]近20轮总进球数分布，选择占比最高的区间：
- 0-1球占比最高 → 比分限定在0:0/1:0/0:1/1:1
- 2-3球占比最高 → 比分限定在1:1/2:1/1:2/2:0/0:2/3:0/0:3
- 4-6球占比最高 → 开放大比分候选

### 维度4: 攻防数据校准（进失球修正）
从[阵容Tab]技术统计提取：
- 主队主场场均进球 vs 客队客场场均失球 → 主队期望进球
- 客队客场场均进球 vs 主队主场场均失球 → 客队期望进球
- 两队期望进球交叉 → 最可能的总进球数

### 维度5: 亚指盘口反推（净胜球约束）
从让球数反推最可能的净胜球区间：
- 让0球 → 平局(净胜0)或1球小胜
- 让0.5球 → 主不败方向(净胜≥0)
- 让1球 → 主胜1球或更多(净胜≥1)
- 让-1球 → 客不败方向(净胜≤1)

### 维度6: 盈亏热度修正（比分概率调整）
- 某方向交易占比>65% + 庄亏/总量>20% → **该方向对应比分降为异常预测**
- 庄盈方向 → 对应比分升为正常预测
- 平局交易占比>70% + 庄亏平局 → 排除平局比分，全部分胜负

### 维度7: 半全场趋势（终场方向）
从[欧指Tab]近20场主客半全数据：
- 主队主场半场胜率>40% → 偏向主队先进球比分
- 客队客场半场不败率>60% → 偏向客队不败比分
- 半全场逆转率 → 判断是否可能出现逆转比分

### 最终合成规则
1. 候选比分经过7个维度逐层过滤，每层排除不匹配的比分
2. 剩余比分按出现频率排序，取前2个为**正常双选**
3. 从被排除的比分中选取2个有依据支撑的为**异常预测**
4. **每个比分必须标注来自哪些维度**（如"[D2+D4] 1:1"）
5. 异常预测若无任何维度支撑 → 标注"[理论可能性]"
4. **亚指唯一依据**：让球结果判定仅依据[战绩Tab]中"亚指"列数据  
   ⚠️ **亚指缺失fallback**：若战绩Tab无亚指列或数据不全 → 降级使用[欧指Tab]指数变化+[盈亏Tab]庄赢方向综合判定，并标注"⚠️亚指数据缺失，让球结果基于欧指+盈亏推导"
5. **禁用模糊词**：删除"可能/大概/倾向"，改用"数据表明/记录显示/统计得出"
6. **⛔ 严禁脚本**：数据获取与预测过程**严禁使用Python/JS脚本**。获取数据只能逐Tab手切+`document.body.innerText`原样读取；预测只能纯逻辑推演，禁止`execute_code`/`terminal`运行建模代码。
7. **逐场串行**：必须**一场一场来**——获取第N场全部Tab数据→立即完成第N场预测→再开始第N+1场。禁止批量预加载多场数据后集中预测。

## ⚡ 让球方向交叉验证规则（基于5月14日复盘）

> 战绩Tab亚指列是让球判定的**第一依据**，但当以下矛盾出现时，必须交叉验证：

### 规则1: 控平陷阱识别
**触发条件**: [欧指Tab]平赔↓家数 ≥ 10家 + 低赔方赔率 ≤ 1.90 + [盈亏Tab]低赔方交易占比 > 60%
**裁决**: 控平家数为**诱平噪音**，真实方向 = 低赔方。让球判定跟随低赔方方向。
**反例**: 4002达曼协作vs吉达联合 — 20家控平 → 表面防平，实际客胜赔1.97+交易67.7% → 客胜
**强制输出**: 识别到控平陷阱时，让球结果必须加注"⚠️控平陷阱已过滤"

### 规则2: 深盘(让≥2球)反直觉规则
**触发条件**: 本场让球 ≥ 2球 + [欧指Tab]主胜升家数 ≥ 15家 + 亚盘降盘家数 ≥ 5家
**常规判断**: 升赔+降盘 = 看衰主队 → 让负
**反直觉裁决**: 若[盈亏Tab]庄家亏损/总交易量 < 10% → **非看衰，而是机构蓄水/诱下盘**。此时让球方向修正为**让平**（卡盘收割为经典操盘手法）。
**反例**: 4005皇马vs奥维耶多 — 升胜21家+降盘9家 → 常规判让负❌；庄亏18K/总613K=3% → 实为诱下盘，结果2-0卡盘让平✅
**强制输出**: 修正为让平时，必须标注"⚠️深盘诱下盘信号，修正为让平"

### 规则3: 亚指列矛盾优先
**触发条件**: [战绩Tab]亚指列赢盘率 > 50% 但盘口支持对方
**裁决**: **盘口优先于赢盘率**。赢盘率仅作辅助参考，不推翻盘口方向。
**强制输出**: "⚠️赢盘率与盘口矛盾，按盘口方向判定"

## 📡 逐场数据获取工作流（单场完整流程）

### 单场获取步骤

```

```

### Tab切换命令（唯一可靠方式）

```javascript
// ⚠️ browser_click(ref) 在Vant UI下不可靠，必须用原生.click()
(function(){
  var d=document.querySelectorAll('.van-tab');
  for(var i=0;i<d.length;i++){
    if(d[i].textContent.includes('欧指')){d[i].click();return'ok';}
  }

```

### 常见故障与兜底

| 故障 | 表现 | 解决 |
|:---|:---|:---|
| **innerText=0** | Tab切换后页面内容丢失 | 重新`browser_navigate`进入详情页，再切Tab |
| **browser_click无效** | 点击ref后Tab未切换 | 改用原生`.click()`（上方命令） |
| **阵容Tab加载慢** | innerText无阵容数据 | 多等1-2秒重试，或跳过阵容标"[阵容数据未获取]" |

> 当预测场次 ≥ 5场时，逐Tab提取完整原始数据会导致会话过长。采用**混合模式**：

1. **列表页提取**：比赛编号/队名/排名/SP/让球数 — 一次性完成
2. **matchID批量匹配**：从北单列表页 `883/danchang/` 通过队名交叉匹配获取所有matchID，比逐场点"分析"快10倍

5. **预测输出**：只展示V2结构化预测结果+关键信号引用，不粘贴原始数据

```python
subprocess.run(["git", "commit", "-m", f"预测: {date_str} 竞足V2预测记录"], check=True)
```

**Step 2 — Push（后台terminal + wait，唯一定时可靠方式）**:
```
terminal(command="cd /root/football-predictions && git push origin main 2>&1; echo EXIT:$?",
         background=true, timeout=300, notify_on_complete=true)
→ process(action='wait', session_id='...', timeout=200)
→ 检查输出含 "EXIT:0" 即成功
```

**多commit逐个推送**：当有多个commit需推送时，逐条commit逐条push，比合并成一条push更可靠（单次payload小，失败可单独重试）。

### 推送内容格式
文件顶部必须包含元数据头：
```markdown
# 竞足预测记录 - {日期}
## 模型: jingzu-prediction-v2
## 场次: {N}场
---
```
后接完整V2输出格式（比分预测/让球结果/模块解析/推理链/风险提示）。

### 赛后复盘推送
比赛结束后，在同目录创建 `{date}-review.md`，包含SPF/让球准确率统计 + 失败分析 + 规则修正记录。
